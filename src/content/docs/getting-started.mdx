This section will walk you through installing Ordo, starting an orchestrator, connecting your first agent, and applying your first state change.

:::note
Content under construction. The full walkthrough lands once the desired-state and secret-distribution surfaces stabilise.
:::

## What you will learn

- Installing the `ordo`, `ordo-orchestrator`, and `ordo-agent` binaries on Linux, macOS, and Windows
- Bootstrapping an orchestrator and creating the first operator identity
- Connecting an agent and approving it from the operator CLI
- Tagging an agent and applying a small declarative state module to it
- Inspecting the audit log and apply history afterwards

## Prerequisites

- A machine to run the orchestrator (Linux, macOS, or Windows)
- One or more machines to run agents — these can be the same machine as the orchestrator while you are exploring
- Network connectivity from each agent to the orchestrator on the configured ports
